# CONCEPT.md — nintendo-switch-web 개편 컨셉

> v0.1 초안 (2026-07-13, 조아진 합의). 개편 4단계(리메이크) 시작 시 이 문서 기준으로 범위 합의 후 v1.0 확정.

## 한 문장 컨셉

**"닌텐도 스위치 2 홈 화면을 웹으로 재현하고, 대표 게임별 페이지가 실제 인게임 같은 인터랙션으로 프로젝트를 보여주는 사이트"**

## 퀄리티 기준 (★ 이번 개편의 핵심 요구)

- "비슷한 분위기" 수준이 아니라, **스위치 홈 화면을 아는 사람이 보면 바로 알아볼 정확도**로 재현한다:
  - 상단 상태바(프로필 아이콘 / 시간 / Wi-Fi / 배터리)
  - 게임 카드 행: 크기·간격·둥근 모서리 비율, 포커스 시 컬러 테두리 하이라이트 + 살짝 확대 + 타이틀 표시
  - 하단 버튼 가이드(ⓐ 결정 / ⓑ 뒤로), 하단 아이콘 메뉴(뉴스/샵/앨범/설정 등은 포트폴리오 메뉴로 치환)
- 게임별 페이지도 "소개 페이지"가 아니라 **그 게임의 인게임 화면처럼** 보이고 움직여야 한다.
- 단, 페이지당 핵심 인터랙션 1개를 매끈하게 (개편_플레이북 3절 원칙 유지).

## 레퍼런스·에셋 워크플로 (저작권 규칙과 세트)

- **참고는 자유, 커밋은 자체 제작만.**
  - Figma 커뮤니티의 Switch UI 킷·실기 스크린샷·인게임 영상은 **치수/색/모션 레퍼런스로만** 사용.
  - repo에 들어가는 에셋은 직접 만든 CSS/SVG/Canvas만. 닌텐도 공식 이미지·스프라이트·음원·영상 커밋 금지.
  - 기존 `images/`의 마리오 ?박스 PNG, 젤다 타이틀 mp4 등은 개편 시 자체 제작물로 교체.
- README에 팬 프로젝트·비상업 학습 목적 명시.
- "공식 에셋 없이 CSS/SVG로 재현" 자체를 README 어필 포인트로 쓴다.

## 페이지별 컨셉 표

| 페이지 | 살릴 아이덴티티 | 핵심 인터랙션 (1개) |
|---|---|---|
| 홈 | 스위치 2 홈 화면 | 방향키/마우스로 카드 포커스 이동, ⓐ(클릭/Enter)로 페이지 진입. 포커스 시 하이라이트·확대 |
| 마리오 | 지상 스테이지 사이드뷰 (하늘·언덕·파이프·벽돌) | **마리오를 조작(←→/스페이스)해 점프로 ?박스를 치면 프로젝트 카드가 튀어나옴.** 박스 개수 = 프로젝트 개수, 친 박스는 빈 박스로 변함, 코인 카운터 연동. 현행 "박스 그리드 나열"은 폐기 |
| 젤다 | 탐험·발견의 분위기 | 스크롤 탐험 연출로 콘텐츠가 발견되듯 나타남 (영상 배경 → 직접 구현 연출로 교체) |
| 기타 | 개편 2단계 진단에서 살릴 것/버릴 것 분류 후 확정 | - |

## 하지 않을 것

- 유튜브 등 외부 콘텐츠 임베드로 때우기
- 닌텐도 공식 에셋 커밋
- 실습 파일(grid생식, 회원가입, 갤럭시워치, trans실습 등)을 홈에 링크 — 개편 1단계에서 repo 밖으로 분리
- 페이지당 어설픈 효과 여러 개 (1개를 매끈하게)
